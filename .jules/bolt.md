## 2024-05-24 - Optimizing Network Latency with asyncio.gather in Retrieval Pipelines
**Learning:** Sequential execution of LLM tool calls and multi-repository searches introduces significant network latency bottlenecks in the `RetrievalPipeline` and `CodeRetrievalPipeline`. The LLM frequently triggers multiple independent tool calls (e.g. searching across symbols, files, annotations) that can and should be processed concurrently to reduce overall wait times. A similar bottleneck occurs when searching across multiple repositories sequentially.
**Action:** Always map over arrays of tool calls or arrays of repositories with `asyncio.gather` for independent network operations (e.g., Pinecone semantic search, Neo4j traversals) instead of using blocking `for` loops. To ensure thread safety when modifying shared lists (e.g. `sources`, `tool_messages`), await `asyncio.gather` and process the array of results sequentially.
