## 2024-05-18 - [Optimization of LLM tool calls and Vector searches in Code Retrieval Pipeline]
**Learning:** LLM tool execution responses and cross-repository vector namespace searches were being executed sequentially within a loop. Using `asyncio.gather` for both the list of tool calls and multi-namespace searches significantly reduces latency without introducing complexity, effectively parallelizing the most time-consuming independent asynchronous I/O paths in the retrieval pipeline.
**Action:** When a pipeline loops to execute LLM tool calls or to perform I/O across multiple data domains/namespaces independently, structure the internal call as a helper `async def` function if state capture is needed, and always wrap the loop with `asyncio.gather(*tasks)` rather than awaiting them sequentially. Ensure the aggregated result array is flattened properly.
